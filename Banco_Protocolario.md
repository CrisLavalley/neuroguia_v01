# NeuroGuía – Banco Protocolario

## Propósito
Este banco se diseñó para que NeuroGuía pueda responder de forma sólida. La lógica propuesta es:

1. Identificar tema, intención, emoción y perfil.
2. Comprimir semánticamente el mensaje libre.
3. Buscar primero en banco protocolario.
4. Responder localmente con:
   - validación breve,
   - lectura funcional,
   - 3 a 5 apoyos concretos,
   - límite no clínico.
5. Personalizar o reescribir cuando haga falta.

## Alcance
Este banco cubre perfiles y condiciones ampliamente reconocidos en marcos de neurodiversidad y condiciones frecuentemente incluidas en marcos amplios o con debate. No pretende cerrar un listado absoluto de "todas las neurodivergencias", pero sí ampliar el espectro de cobertura práctica para familia, escuela y autocuidado.

## Recomendaciones de integración
- Reemplazar el archivo base de protocolos por `protocol_bank_expanded.json`.
- Hacer que el orquestador consulte primero `general_query_answers` y `conditions`.
- Usar `combination_protocols` cuando detecte co-ocurrencias.
- Humanizar, resumir o personalizar, no para construir la respuesta desde cero.

## Nota metodológica
Esta estructura fortalece la defensa de tesis porque sitúa el valor del proyecto en:
- la arquitectura híbrida,
- la compresión semántica local,
- el banco curado de conocimiento.
